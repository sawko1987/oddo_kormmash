# Не создавать инструкции автоматически

## Основное правило

**НЕ создавайте файлы с инструкциями (README.md, INSTALL.md, GUIDE.md и т.п.) автоматически**, если пользователь явно не попросил об этом.

## Когда НЕ создавать инструкции

- После выполнения задачи (установка модуля, настройка и т.д.)
- При решении проблемы
- При выполнении команды
- При обновлении кода
- При настройке конфигурации

## Когда можно создавать инструкции

Только если пользователь **явно попросил**:
- "Создай инструкцию..."
- "Напиши README..."
- "Сделай документацию..."
- "Создай файл с описанием..."

## Альтернатива

Вместо создания файлов с инструкциями:
- **Объясните кратко в ответе**, что было сделано
- **Укажите следующие шаги** прямо в сообщении
- **Покажите команды** для выполнения, если нужно

## Примеры

### ❌ Плохо
```
[Выполняет задачу]
[Создает файл INSTRUCTIONS.md с подробными инструкциями]
```

### ✅ Хорошо
```
[Выполняет задачу]
Готово! Модуль установлен. Перезапустите Odoo сервер командой:
python3 odoo-bin -c odoo.conf
```

## Исключения

Можно создавать инструкции только если:
1. Пользователь явно попросил
2. Это часть явно запрошенной задачи (например, "создай документацию проекта")
