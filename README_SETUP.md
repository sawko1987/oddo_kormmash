# Быстрый старт: Настройка Odoo для снабженца

## Обзор

Этот документ содержит краткую инструкцию по началу работы с настройками Odoo для снабженца машиностроительного предприятия.

## Шаг 1: Проверка модулей

### Вариант 1: Упрощенная проверка (без подключения к БД)

Проверяет наличие модулей в файловой системе:
```bash
cd "/home/sawko1987/ Odoo"
python3 check_installed_modules_simple.py
```

### Вариант 2: Полная проверка (требует подключения к БД)

Проверяет установленные модули в базе данных:
```bash
cd "/home/sawko1987/ Odoo"
# Активируйте виртуальное окружение (psycopg2 уже установлен там):
source venv/bin/activate
python3 check_installed_modules.py
```

**Примечание:** Скрипт автоматически использует peer authentication (Unix socket) если в `odoo.conf` не указан `db_host`.

Если модули не установлены, используйте:
```bash
./install_modules.sh
```

Или установите через интерфейс Odoo: **Apps → [название модуля] → Install**

## Шаг 2: Настройка базовых параметров

1. Войдите в Odoo как администратор
2. Перейдите в **Settings → Purchase**
3. Настройте:
   - ✅ Purchase Order Approval (двухэтапное утверждение)
   - ✅ Lock Confirmed Orders (блокировка заказов)
   - ✅ Warnings (предупреждения)
   - ✅ Receipt Reminder (напоминания о поставках)
   - ✅ Purchase Agreements (тендеры и соглашения)

## Шаг 3: Создание поставщиков

1. Перейдите в **Purchase → Vendors**
2. Создайте поставщиков вручную или импортируйте из CSV
3. Пример CSV файла: `examples/supplier_template.csv`

## Шаг 4: Настройка правил закупок

1. Перейдите в **Inventory → Configuration → Reordering Rules**
2. Создайте правила для критичных материалов
3. Подробная инструкция: `examples/procurement_rules_setup.md`

## Шаг 5: Настройка отслеживания

1. Откройте карточки критичных продуктов
2. На вкладке **Inventory** установите **Tracking**:
   - **Serial Numbers** для критичных компонентов
   - **Lots** для материалов с сертификатами
   - **No Tracking** для стандартных материалов
3. Подробная инструкция: `examples/tracking_setup.md`

## Шаг 6: Настройка интеграции с MRP (если используется)

1. Убедитесь, что модуль `mrp` установлен
2. Модуль `purchase_mrp` установится автоматически
3. Настройте продукты и спецификации
4. Подробная инструкция: `examples/mrp_integration_setup.md`

## Шаг 7: Настройка тендеров

1. Активируйте модуль Purchase Requisition в настройках
2. Создайте рамочные соглашения для регулярных закупок
3. Используйте тендеры для крупных закупок
4. Подробная инструкция: `examples/tenders_setup.md`

## Шаг 8: Настройка отчетности

1. Перейдите в **Purchase → Reporting**
2. Настройте дашборды для быстрого доступа
3. Создайте сохраненные поиски для регулярных отчетов
4. Подробная инструкция: `examples/reports_setup.md`

## Шаг 9: Настройка автоматизации

1. Настройте напоминания о поставках в карточках поставщиков
2. Проверьте email-шаблоны в **Settings → Technical → Email**
3. Настройте автоматические отчеты при необходимости
4. Подробная инструкция: `examples/automation_setup.md`

## Полная документация

- `check_installed_modules.py` - проверка установленных модулей
- `setup_purchase_config.py` - автоматическая настройка параметров
- `install_modules.sh` - установка модулей
- `PURCHASE_SETUP_GUIDE.md` - полное руководство
- `examples/` - примеры и инструкции по настройке

## Поддержка

При возникновении проблем:
1. Проверьте логи: `./var/odoo.log`
2. Убедитесь, что все модули установлены
3. Проверьте права доступа пользователей
4. Обратитесь к документации Odoo: https://www.odoo.com/documentation
