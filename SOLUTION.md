# Решение проблемы запуска Odoo

## Проблемы обнаружены

1. ✅ **Исправлено**: Ошибка `OrderedSet.copy()` - метод добавлен
2. ✅ **Исправлено**: Установлен `pyOpenSSL`
3. ⚠️ **Требует внимания**: Ошибка парсинга XML при инициализации

## Решение

### Вариант 1: Запуск с подробным логированием для диагностики

```bash
python odoo-bin -c odoo.conf -d odoo --init=base --stop-after-init --log-handler odoo.tools.convert:DEBUG
```

Это покажет полную ошибку парсинга XML.

### Вариант 2: Инициализация без демо-данных

```bash
python odoo-bin -c odoo.conf -d odoo --init=base --stop-after-init --without-demo=all
```

### Вариант 3: Использование веб-интерфейса для инициализации

1. Запустите Odoo без инициализации:
   ```bash
   python odoo-bin -c odoo.conf
   ```

2. Откройте http://localhost:8069

3. В веб-интерфейсе выберите "Create database" и настройте базу данных

### Вариант 4: Использование Python 3.12 (рекомендуется)

Odoo 19.0 официально поддерживает Python 3.12, а не 3.14. Рекомендуется:

1. Установить Python 3.12
2. Создать новое виртуальное окружение:
   ```bash
   python3.12 -m venv venv312
   venv312\Scripts\activate  # Windows
   pip install -r requirements.txt
   pip install psycopg2-binary pyOpenSSL PyPDF2
   ```
3. Запустить инициализацию:
   ```bash
   python odoo-bin -c odoo.conf -d odoo --init=base --stop-after-init
   ```
