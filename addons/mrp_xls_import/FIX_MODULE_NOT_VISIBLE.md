# Решение проблемы: Модуль не виден в Apps

## Проблема
Модуль `mrp_xls_import` создан, но не отображается в списке Apps в Odoo.

## Решение 1: Обновить список модулей в Odoo

### Через веб-интерфейс:
1. Откройте страницу **Apps** (Приложения)
2. В правом верхнем углу найдите кнопку **"Update Apps List"** (Обновить список приложений)
3. Нажмите на неё
4. Дождитесь завершения обновления
5. Обновите страницу (F5)

### Через командную строку:
```bash
# Остановите Odoo (если запущен)
# Затем выполните:
python odoo-bin -c odoo.conf -d odoo --stop-after-init --update=all
```

## Решение 2: Проверить фильтры в Apps

1. На странице **Apps** найдите фильтры вверху
2. **Снимите фильтр "Apps"** (если установлен)
3. Выберите **"All"** или **"Custom"** в фильтре
4. В поиске введите: `mrp_xls` или `XLS Import`
5. Модуль должен появиться

## Решение 3: Перезапустить Odoo

1. Остановите Odoo сервер (Ctrl+C в терминале)
2. Запустите заново:
   ```bash
   python odoo-bin -c odoo.conf
   ```
3. Обновите страницу Apps в браузере (Ctrl+F5)

## Решение 4: Проверить права доступа

Убедитесь, что вы вошли как **администратор** (пользователь с правами администратора).

## Решение 5: Прямая установка через команду

Если модуль все еще не виден, попробуйте установить его напрямую:

```bash
python odoo-bin -c odoo.conf -d odoo -i mrp_xls_import --stop-after-init
```

Затем перезапустите Odoo:
```bash
python odoo-bin -c odoo.conf
```

## Решение 6: Проверить логи на ошибки

Проверьте логи Odoo на наличие ошибок при загрузке модуля:

```bash
# Просмотр последних строк лога
tail -100 var/log/odoo.log
```

Ищите ошибки, связанные с `mrp_xls_import`.

## Решение 7: Проверить структуру модуля

Убедитесь, что модуль находится по пути:
```
c:\oddo_kormmash\addons\mrp_xls_import\
```

И содержит все необходимые файлы (проверено - все файлы на месте).

## Быстрая проверка

Выполните эту команду для проверки, видит ли Odoo модуль:

```bash
python odoo-bin -c odoo.conf -d odoo --stop-after-init --log-level=debug 2>&1 | grep -i "mrp_xls"
```

Если модуль найден, вы увидите строки с его именем в логах.

## Если ничего не помогает

1. Проверьте, что в `odoo.conf` указан правильный путь:
   ```
   addons_path = ./addons,./odoo/addons
   ```

2. Убедитесь, что модуль не имеет синтаксических ошибок в `__manifest__.py`

3. Попробуйте создать тестовый простой модуль для проверки, что Odoo вообще видит кастомные модули
