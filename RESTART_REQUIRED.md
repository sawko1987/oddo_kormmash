# ⚠️ ТРЕБУЕТСЯ ПЕРЕЗАПУСК СЕРВЕРА

## Проблема

Ошибка: `"res.users"."auth_passkey_key_ids" field is undefined`
Ссылка "Use a Passkey" отображается на странице входа.

## Что сделано

1. ✅ Модуль `auth_passkey` переименован в `auth_passkey.disabled`
2. ✅ Все данные модуля удалены из базы данных
3. ✅ Все поля, модели, представления удалены
4. ✅ Представление `web.login_oauth` обновлено

## ⚠️ ВАЖНО: Перезапуск обязателен!

Все изменения применены к базе данных, но **сервер все еще использует старые закэшированные данные**. 

### Шаг 1: Остановите сервер

Нажмите `Ctrl+C` в терминале, где запущен Odoo.

### Шаг 2: Запустите сервер снова

```bash
source venv312/Scripts/activate
python odoo-bin -c odoo.conf
```

### Шаг 3: Очистите кэш браузера

- Нажмите `Ctrl+Shift+R` (жесткая перезагрузка)
- Или откройте в режиме инкогнито

### Шаг 4: Проверьте

Откройте: http://localhost:8069/web/login

**После перезапуска сервера:**
- ✅ Ссылка "Use a Passkey" исчезнет
- ✅ Ошибка `auth_passkey_key_ids` исчезнет
- ✅ Вы сможете войти в систему
- ✅ Настройки будут работать

## Почему нужен перезапуск?

Odoo кэширует:
- Загруженные модули
- Представления (views)
- Модели и поля
- JavaScript-код

Без перезапуска сервер продолжает использовать старые данные из памяти.

## После перезапуска

Вы сможете:
- ✅ Войти в систему (admin/admin)
- ✅ Открыть настройки без ошибок
- ✅ Использовать все функции Odoo

**Перезапустите сервер сейчас!**
